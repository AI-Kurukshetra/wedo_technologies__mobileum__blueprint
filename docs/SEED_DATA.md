# Seed Data — TeleGuard Pro

Seed data enables:

- a demo org and users
- sample CDR imports/records
- starter fraud rules
- sample alerts and a case

This can be implemented as:

- SQL migration in Supabase (for non-auth tables)
- a one-time script (recommended) that:
  - creates users via Supabase Admin API
  - inserts tenant data via service role

## 1) Demo Organization

- Org:
  - name: `Acme Telco`
  - slug: `acme`

## 2) Demo Users (suggested)

Create these users and add memberships for the demo org:

- `admin@acme.example` (role: `admin`)
- `manager@acme.example` (role: `manager`)
- `analyst@acme.example` (role: `analyst`)
- `readonly@acme.example` (role: `read_only`)

Password strategy for local/dev:

- use a fixed password from env, e.g., `SEED_USER_PASSWORD`

## 3) Demo CDR Imports

Create 2 imports:

1. `cdr_acme_normal.csv` — typical traffic
2. `cdr_acme_attack.csv` — traffic spike to high-risk destinations

For each import:

- insert a `cdr_imports` row with `status='processed'`
- insert ~5k–20k `cdr_records` rows across the last 7 days

### 3.1 Minimal CDR CSV Template (MVP)

Assume these columns (fixed mapping):

- `call_start_at` (ISO string)
- `duration_seconds` (int)
- `a_party` (string)
- `b_party` (string)
- `destination_country` (ISO2)
- `destination_prefix` (string)
- `account_id` (string)
- `carrier_id` (string)
- `answer_status` (answered/failed/no_answer)
- `revenue_amount` (decimal)
- `cost_amount` (decimal)
- `currency` (string)

### 3.2 Data Generation Guidance

Normal traffic:

- 70% destination_country = `US`
- 10% = `GB`, 5% = `CA`, 5% = `AU`, 10% spread across others
- answered_rate ~ 75–90%

Attack traffic (fraud simulation):

- For a single `account_id` (e.g., `ACME-001`), in a 15-minute window:
  - call_count spikes to 200–500
  - destination_country includes `NG`, `PK`, `GH` (example “high risk”)
  - shorter durations and higher failed rate (optional)

## 4) Starter Fraud Rules (MVP)

Insert 3–5 rules, each with a version snapshot:

### 4.1 Rule: High International Call Volume (by account)

- status: enabled
- severity: high
- window_minutes: 15
- dimension_type: `account_id`
- filters: destination_country not in [`US`]
- thresholds: call_count >= 200
- dedup_minutes: 60

### 4.2 Rule: High Failed Call Rate (by carrier)

- status: enabled
- severity: medium
- window_minutes: 30
- dimension_type: `carrier_id`
- thresholds:
  - call_count >= 100
  - failed_rate >= 0.5

### 4.3 Rule: Revenue Spike (by destination country)

- status: enabled
- severity: high
- window_minutes: 60
- dimension_type: `destination_country`
- thresholds: total_revenue >= 5000

### 4.4 Rule: Unusual Short-Duration Burst (by a_party)

- status: disabled (so users can enable)
- severity: low
- window_minutes: 10
- dimension_type: `a_party`
- thresholds:
  - call_count >= 300
  - avg_duration_seconds <= 10 (if implemented)
